# Voice DNA

## Overview
Analyzes writing samples to extract voice patterns — tone, rhythm, vocabulary, sentence structure, and stylistic quirks — and creates a reusable voice profile. Prevents AI from "correcting" intentional stylistic choices.

## When to Use This Skill
- Before starting any writing project where voice consistency matters
- When onboarding a new writer or brand voice
- When AI outputs sound generic and need personality injection
- When switching between multiple brand voices

## Process

### Phase 1: Sample Collection
Collect 3-5 representative writing samples (500+ words each). Choose pieces that best represent the target voice.

**Selection criteria:**
- Published and approved (not drafts)
- Written by the target voice owner
- Span different topics but same voice
- Include both strong and subtle stylistic moments

### Phase 2: Pattern Extraction
Analyze each sample across these dimensions:

1. **Sentence Structure**: Average length, variation pattern, fragment usage, compound vs simple
2. **Vocabulary Level**: Formal/informal ratio, jargon density, preferred synonyms
3. **Rhythm**: Short-long sentence alternation, paragraph cadence, use of lists vs prose
4. **Tone Markers**: Humor style, directness level, hedging language, confidence signals
5. **Signature Moves**: Recurring phrases, opening patterns, transition habits, closing style
6. **Rule Breaks**: Intentional grammar violations, style guide deviations, quirks to preserve

### Phase 3: Profile Creation
Compile findings into a structured voice profile document:

```markdown
# Voice Profile: [Name/Brand]

## Core Voice Attributes
- **Tone**: [e.g., conversational but authoritative]
- **Formality**: [1-10 scale]
- **Humor**: [type and frequency]
- **Directness**: [1-10 scale]

## Sentence Patterns
- Average length: [X words]
- Preferred structures: [list]
- Signature openings: [examples]

## Vocabulary
- Preferred terms: [list of go-to words]
- Avoided terms: [words this voice never uses]
- Jargon approach: [embraces/avoids/defines]

## Intentional Rule Breaks
- [List stylistic choices that should NOT be corrected]

## Example Passages
[3-5 short excerpts that capture the voice perfectly]
```

### Phase 4: Validation
Test the profile by generating new content and comparing against original samples. Adjust until the output is indistinguishable from the source voice.

## Key Rules
- Never average out distinctive voice traits — preserve extremes
- Intentional rule breaks are features, not bugs
- Update the profile as voice evolves over time
- Keep sample passages in the profile for quick reference
- Voice profiles are living documents, not one-time outputs
