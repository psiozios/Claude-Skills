# Micro Experiments

## Overview
Helps design low-stakes career or product exploration experiments. Defines hypothesis, minimum viable test, success criteria, time-box, and decision framework. Useful for career transitions, new market entry, or feature validation without full commitment.

## When to Use This Skill
- Exploring a career change without quitting your job
- Testing a new product idea before building
- Validating a market hypothesis cheaply
- Making a decision under uncertainty

## Process

### Step 1: Define the Question
- What are you trying to learn?
- What decision will this inform?
- What's the smallest test that would give you useful signal?

### Step 2: Design the Experiment

```markdown
## Experiment: [Name]

### Hypothesis
If [action], then [expected outcome], because [reasoning].

### Minimum Viable Test
[The simplest, cheapest way to test this hypothesis]

### Success Criteria
- **Strong signal**: [What would make this a clear yes]
- **Weak signal**: [What would be encouraging but inconclusive]
- **Failure signal**: [What would make this a clear no]

### Time Box
- Duration: [1 week / 2 weeks / 1 month]
- Hours invested: [maximum]
- Money invested: [maximum]

### Decision Framework
- If strong signal → [next step]
- If weak signal → [iterate or expand test]
- If failure signal → [pivot or stop]
```

### Step 3: Execute
- Set a calendar reminder for the end date
- Track observations as you go (don't rely on memory)
- Resist the urge to extend the time-box
- Talk to real people, don't just think about it

### Step 4: Evaluate
- Review against success criteria (not feelings)
- Document what you learned
- Make the decision you committed to
- Design the next experiment if needed

## Key Rules
- The experiment must be small enough to feel safe
- Define success criteria BEFORE starting (prevents moving goalposts)
- Time-box everything — open-ended experiments never end
- One variable at a time — don't test everything at once
- The goal is learning, not proving you're right
