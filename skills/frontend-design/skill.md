# Frontend Design

## Overview
Creates distinctive, production-grade UI designs and interfaces with bold aesthetic directions. Emphasizes typography choices, spatial composition, color theory, and interaction patterns. Outputs design specifications or working HTML/CSS prototypes.

## When to Use This Skill
- Designing a new product interface from scratch
- Redesigning an existing UI that feels generic
- Creating a design system or component library
- Building a high-fidelity prototype

## Process

### Step 1: Design Brief
Gather requirements:
- Product type and target users
- Emotional response to evoke (trust, excitement, calm, urgency)
- Reference designs (likes and dislikes)
- Technical constraints (framework, responsive requirements)
- Brand guidelines (if they exist)

### Step 2: Design Direction
Establish the aesthetic:

**Typography:**
- Heading typeface: character, weight, sizing scale
- Body typeface: readability, pairing harmony
- Type scale: modular scale ratio (1.25, 1.333, 1.5)
- Line heights and letter spacing

**Color:**
- Primary palette (3 colors max for main UI)
- Semantic colors (success, warning, error, info)
- Neutral scale (8-10 shades of gray)
- Contrast ratios (WCAG AA minimum)

**Spatial Composition:**
- Grid system (columns, gutters, margins)
- Spacing scale (4px or 8px base unit)
- Component density (compact, comfortable, spacious)
- White space strategy

### Step 3: Component Design
Build core components:
- Buttons (primary, secondary, ghost, destructive)
- Forms (inputs, selects, checkboxes, validation states)
- Cards and containers
- Navigation patterns
- Typography styles (H1-H6, body, caption, overline)

### Step 4: Output
- Design tokens (CSS custom properties)
- Working HTML/CSS prototype
- Component documentation
- Responsive behavior specifications

## Key Rules
- Typography is 90% of design — get the type right first
- Limit yourself to 2 typefaces maximum
- Every spacing value should come from the scale (no magic numbers)
- Design for the smallest screen first, then scale up
- Accessibility is not optional — WCAG AA minimum on all text
